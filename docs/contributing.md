## 参与编辑

SJTU Wiki 是一个由学生主导建设的校园百科平台，旨在系统整合上海交通大学校内与学习、生活、事务办理等息息相关的信息资源——从选课规则详解、食堂窗口测评，到正版软件使用、奖学金申请流程、宿舍生活小贴士，都能在这里找到交大人自己的经验与答案。我们希望通过构建便捷、高效的信息导航系统，将它打造成一个不断生长、众人共创的校园知识库。

如果你也是校园百事通，或是对交大生活的某一方面特别精通，又或者善于梳理、总结各类零散信息，愿意把自己的经验传递给更多交大人，我们都欢迎你加入 SJTU Wiki，共同建设属于所有交大人的百科。

## 工作流

### 通过 Github 贡献

要将您的文章投稿到 SJTU Wiki，您需要 Fork 一份代码仓，并在自己的代码仓中进行编辑，完成后提交 [Pull Request](https://github.com/SJTU-Geek/SJTU-Wiki/pulls) 来申请合入代码。至少两名管理员同意后，您的代码将会合入到本代码仓，并自动更新到预览网站。

### 通过在线问卷贡献

我们还为不熟悉 GitHub 的用户准备了问卷投稿方式，您可以直接提交文本或上传 Word / Markdown 文档参与 SJTU Wiki 建设：<https://ssc.sjtu.edu.cn/f/f787ebb3>

## 内容规则

### 立场
SJTU Wiki 致力于成为一个**绝对客观**的校园知识平台，我们希望您能以**书面、客观的语言**描述上海交通大学的客观事实。因此，请贡献者在创作和编辑时遵循以下准则：

- **遵循法律和校规**：这是内容平台的底线，任何违法违规的内容都不会被允许。请勿发表涉及“擦边”“灰色地带”的观点和言论。
- **优先采信官方来源**：对于制度、政策、流程类内容，优先选用学校官方发布的原文、通知及链接。我们鼓励贡献者通过广泛的群众力量，寻找并标注官方原文的出处，确保信息的权威性和可追溯性。
- **描述应去主观化**：我们不排斥个人经验分享，但所有主观经验类内容均需经过核准后方可发布。在描述时，应尽可能剥离主观情绪，以“陈述事实事件”的方式呈现经验，避免出现任何带有拉踩、立场对立倾向的表达。
- **不煽动对立风气**：我们理解同学们希望传递“避坑”经验的好意，但 SJTU Wiki 不是随意发泄负面评价的平台。我们不提倡“避雷”“避坑”的叙事方向——这种表达本身容易指向对课程、机制或个人的单纯负面评价，助长不良风气。


我们欢迎您**新增**对上海交通大学师生的学习生活有较强帮助的内容。我们欢迎您**修改** SJTU Wiki 上的错误内容、过时内容和表述欠佳的内容。

#### 署名
您可以在文章开头 author 字段处进行署名，请**使用恰当的昵称**。

请使用 Markdown 的无序列表语法，例如：
```
---
title: 贡献指南
order: 85
author: 
 - SJTU Wiki 编辑部
 - 东中院猫猫
 - 思源湖底
---
```

#### 引用
若是引用了他人创作的内容，请直接在段落中或文末的参考章节中**标注**其来源，并给出**对应链接**。此类链接还需遵守下一节“链接”中的规定。

### 链接
我们允许在 SJTU Wiki 上放置来自政府机构和事业单位的官方网站和文章链接。

对于非官方发布的内容、由用户自行编写的内容（如水源帖子、水源文档等），请在放出原文链接的同时在链接后标出：`<sup>[非官方内容，请谨慎参考]</sup>`。主观性较强的经验类内容也建议在首句后带上此标签。

对于外部网站推荐，请提交可靠的站点（请核查站点的可访问性、稳定性、客观性）上的链接。请注意您提交的链接有被拒绝的可能。

### 时效性

对于时效性较强的内容，请额外提醒读者注意时效性，并给出获取最新信息的方式。

SJTU Wiki 对于过时内容的处理规则：
- 影响力持续至今的事情，尽管可能已经过时，但为了因果关系完整，仍应予以保留。^[举例：2024 年，学校转专业政策从限制两次变更为不限次数，由于 23 级本科生尚未毕业，旧政策仍具有影响力。在 23 级本科生毕业后且该政策持续稳定，将旧政策相关表述按下方规则处理]
- 已无影响力，但影响力消失不足两年的，予以折叠、标注过时（使用 [VitePress 的折叠语法](https://vuejs.github.io/vitepress/v1/zh/guide/markdown#custom-title)）。
- 已无影响力，影响力消失满两年的，予以删除。

## 编辑语法

SJTU Wiki 目前采用 [VitePress](https://vuejs.github.io/vitepress/v1/zh/guide/what-is-vitepress) 软件，将 Markdown 文件转换为一篇篇文章。

Markdown 语言非常简单易上手，如果您不了解什么是 Markdown，可以参考[这篇 Markdown 教程](https://101.lug.ustc.edu.cn/Appendix/markdown)。请特别注意，在 Markdown 中需要输入回车两次来显示换行。

请在文章最前面的头部 title 字段中输入文章标题，正文部分章节标题从 H2（两个井号）开始。

## 架构规范

如果您在 GitHub 进行贡献，请在提交 PR 时注意以下架构规范。

### 目录

如果您希望新增目录，请创建对应的文件夹并在其中放置 `index.md` 文件。如果您希望新增文章，在对应文件夹中创建即可，在文档开头填写 `title` 和 `order` 字段，侧边栏会自动识别并显示新文章。

### 图片

请将所有图片存放在文章所在目录的 `./images` 目录下，如 `/undergraduate-study/study-plan/category.md` 页面用到的图片请放在 `/undergraduate-study/study-plan/images` 目录中。请将图片重命名为**有意义的文件名**，使用**下划线**连接英文单词。

### 附件

请将所有附件存放在文章所在目录的 `./files` 目录下。请将附件重命名为**有意义的文件名**。

## 积分体系
为了感谢您对 SJTU Wiki 建设做出的贡献，您的每次对 SJTU Wiki 的有效编辑都将累计积分，并**可 1:1 兑换成[传承·交大](https://share.dyweb.sjtu.cn/user/points/redeem)的积分**。

- 积分明细可在[思源极客协会 Wiki 贡献中心](https://geek.sjtu.edu.cn/products/wiki/contrib)查询
- 贡献中心可绑定 GitHub 账号，系统自动合并您通过 GitHub 账号提交、合并的 Pull Request 对应的积分

SJTU Wiki 贡献者发放积分的总则如下：

- 新增一篇文章：约 50 分
- 新增一个章节：约 10 分
- 新增一个段落：约 1~3 分

积分细则：
- 以上文章/章节/段落的规模可参考文章《奖助学金》。可获得50积分的文章，其参考字数为 2000 字，计算时减去纯引用内容的字数。
- 修正和更新内容的分数约为新增内容的 60%。
- 内容详细程度和篇幅较突出的，可获得最多 50% 积分加成。
- 部分极小修改可能无法获得积分。
- 如果对他人的经验分享进行排版后放进 SJTU Wiki，请取得原作者授权。搬运者可获得 30% 的贡献分数，原作者可声明并领取剩余积分。
